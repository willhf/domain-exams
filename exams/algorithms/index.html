Exam Scope

This examination assesses mastery of fundamental algorithms and data structures at the undergraduate level, with emphasis on correctness and runtime analysis.

No programming language specifics are required. All presentation of algorithms is expected to be in pseudocode (or the language of choice).

Definitions of core concepts and algorithms will not be provided on the exam. Candidates are expected to know them.

Candidates are expected to reason formally about algorithm correctness.

The scope described here is subject to refinement.

Asymptotic Analysis

  • Formal definitions of Big-O, Θ, and Ω notation
  • Calculation of worst-case runtime analysis
  • Solving recurrences (substitution, recursion tree, Master Theorem)
  • Amortized Analysis

References: CLRS (Ch. 2-4, 14); Sedgewick & Wayne (§1.4)

Algorithm Strategies

  • Divide and conquer
  • Greedy algorithms
  • Dynamic programming

References: CLRS (Ch. 4, 14, 15); Sedgewick & Wayne (Ch. 2.2, 2.3)

Data Structures

  • Arrays, Stacks, Queues, Linked Lists
  • Heaps and Priority Queues
  • Binary Trees, Red-Black Trees
  • Hash Tables
  • B-Trees

Candidate are expected to be able to implement (and understand the runtime analysis of) any of the following operations for each of these data structures: insert, delete, search, min, max, and enumerate.

References: CLRS (6, 10-13, 18); Sedgewick & Wayne (1.3, 3.2-3.4, 6)

Sorting Algorithms

  • Bubble Sort, Selection Sort, Insertion Sort
  • Merge Sort, Quick Sort, Heap Sort, Shell Sort
  • Radix Sort, Counting Sort, Bucket Sort
  • Median and Order Statistics

Candidates are expected to understand the runtime analysis (average and worst-case), memory usage, stability, and other features which distinguish these algorithms from one another.

References: CLRS (Ch. 2, 6-9); Sedgewick & Wayne (Ch. 2)

Graph Algorithms

  • Depth-first search and breadth-first search
  • Topological sort
  • Strongly connected components
  • Minimum Spanning Trees, Kruskal's and Prim's algorithms
  • Shortest Paths, Bellman-Ford and Dijkstra's algorithms

References: CLRS (Ch. 20-22); Sedgewick & Wayne (Ch. 4)

String Algorithms

  • Longest Common Subsequence
  • Levenshtein Distance, Wagner-Fischer algorithm
  • Substring Search: Finite Automata, Rabin-Karp and Knuth-Morris-Pratt algorithms
  • Tries, Suffix Arrays

References: CLRS (Ch. 14.4, 32); Sedgewick & Wayne (5, 6)

Out of Scope

The following topics are not covered in the exam.

  • Randomized Algorithms
  • Number-theoretic, Cryptographic, and Hash Construction algorithms
  • NP-Completeness and Complexity Classes
  • Parallel and Distributed Algorithms
  • Quantum Algorithms
  • Linear Programming
  • Data Compression
  • Algorithms for machine learning or artificial intelligence

Reference Texts

While the exam scope is defined independently of any single textbook, candidates are expected to be comfortable with material as treated in the following canonical undergraduate texts.

These texts collectively define the level and depth expected on the exam. Familiarity with one or more of them is strongly recommended.

  • CLRS — Introduction to Algorithms (Cormen, Leiserson, Rivest, Stein)
  • Sedgewick & Wayne — Algorithms

Exam Format

  • Exam has a time limit of 2.5 hours
  • Exam is written - booklets, pens, and pencils are provided, but you are welcome to bring your writing instrument of choice
  • Closed Book - Notes, phones, and all devices are prohibited
  • Restroom and water breaks are permitted, but do not alter the time limit
